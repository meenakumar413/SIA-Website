Git & GitHub

A Beginner’s Guide to Version Control & Collaboration

Visual: Git & GitHub logos with a clean tech-themed background

Speaker Notes: Welcome! Today we’ll break down Git and GitHub in simple terms using real-world analogies and practical commands.

What Are Git & GitHub?

  • Git: Tracks changes in your project (version control)
  • GitHub: Online platform to store, share, and collaborate on Git projects

Visual: Split screen showing Git as a timeline and GitHub as a cloud

Speaker Notes: Think of Git as a time machine for your code, and GitHub as the place where teams collaborate.

Why Use Git?

  • Saves every change you make
  • Lets you undo mistakes
  • Helps manage multiple versions

Visual: Comic-style “Oops!” moment with Git restoring a previous version

Speaker Notes: Ever broken something and wished you could go back? Git lets you do exactly that.

Why Use GitHub?

  • Share code with others
  • Work together on projects
  • Keep everything organized

Visual: Team of developers pushing code to a shared repo

Speaker Notes: GitHub is where collaboration happens. It’s like Google Docs for code.

Essential Git Commands

CommandPurpose
git initStart a Git project
git statusCheck current changes
git addStage changes
git commitSave changes with a message
git branchCreate a new version path
git checkoutSwitch between branches
git mergeCombine branches
git pushUpload to GitHub
git pullDownload from GitHub

Speaker Notes: These are the 9 commands you’ll use 90% of the time. Master them and you’re good to go.

Connecting to GitHub

  • Create a GitHub account
  • Link your local project
  • Push your code online

Visual: GitHub repo with commit history

Speaker Notes: Once your project is ready, push it to GitHub to share and collaborate.

Real-World Workflow

  • Local changes → Git tracking
  • Push to GitHub → Team collaboration
  • Pull updates → Stay in sync

Visual: Flowchart of developer workflow

Speaker Notes: This is how developers around the world build websites, apps, and APIs together.
