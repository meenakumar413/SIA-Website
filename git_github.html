Git & GitHub

A Beginner’s Guide to Version Control & Collaboration

Welcome! Today we’ll break down Git and GitHub using real-world analogies and practical commands.

What Are Git & GitHub?

  • Git: Tracks changes in your project (version control).
  • GitHub: Online platform to store, share, and collaborate on Git projects.

Think of Git as a time machine for your code, and GitHub as the cloud where teams work together.

Why Use Git?

  • Saves every change you make.
  • Lets you undo mistakes easily.
  • Manages multiple versions of your project.

Ever broken something and wished you could go back? Git lets you do exactly that.

Why Use GitHub?

  • Share code with others.
  • Collaborate on projects in real time.
  • Keep everything organized and versioned.

GitHub is where collaboration happens — like Google Docs for code.

Essential Git Commands

CommandPurpose
git initStart a Git project
git statusCheck current changes
git addStage changes
git commitSave changes with a message
git branchCreate a new version path
git checkoutSwitch between branches
git mergeCombine branches
git pushUpload to GitHub
git pullDownload from GitHub

These are the 9 commands you’ll use 90% of the time.

Connecting to GitHub

  1. Create a GitHub account.
  2. Initialize your local Git project.
  3. Link your repository using git remote add origin.
  4. Push your project online using git push -u origin main.

Once your project is ready, push it to GitHub to share and collaborate.

Real-World Workflow

  1. Make local changes → Git tracks them.
  2. Push to GitHub → Team collaborates.
  3. Pull updates → Stay in sync.

This is how developers across the world build and manage projects together.

Final Takeaways

  • Git = Version control on your computer.
  • GitHub = Collaboration in the cloud.
  • Together they make teamwork seamless and efficient.

Version control is the foundation of modern software development.
